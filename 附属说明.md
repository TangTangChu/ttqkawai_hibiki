# ttqkawai_hibiki - 核心 UX/UI 设计与交互规范架构

（注：本文档基于全站所有页面（如 properties、about、archives）、组件、布局及设计文档严格深度提取，去除了 `ArticleBlock.vue` 的特例交互机制）

## 一、核心视觉基调与形状法则 (Core Visual & Shape)

1. **设计语言**：可爱、简约、灵动、注重交互与微动画效果。
2. **边缘与形状**：偏向于**极简的圆角矩形**。
    - 绝大多数容器（包括按钮、卡片、导航栏块、`favorites.vue` 的图片媒体层等）强制使用 `rounded-xl` 或 `rounded-2xl`，仅 `AnriButton.sm` 等极小交互部件为 `rounded-lg`。
    - 包含比例的缩放媒体控制如 `favorites.vue` 中 `aspect-105/148` 与 `aspect-square`，内嵌 `rounded-xl overflow-hidden` 规范。
    - 排斥过度使用的完全胶囊形变（Pill-shaped，如 `rounded-full` 仅在头像、分页圆点、`Timeline` 左侧圆点中使用）。
3. **层次与阴影隔离**：
    - 彻底抛弃复杂的盒子阴影（Box-shadows）。必须严格克制使用，非必要不使用，需要时仅能使用极浅阴影。
    - 彻底杜绝发光效果、浮色描边。过度包装元素（什么都丢卡片里）是被绝不被允许的。
    - 层级关系的强调完全依赖 **色块堆叠、背景透明度叠加 (opacity / `/5`, `/10`, `/20`) 和 `backdrop-blur` (高斯模糊)**。

## 二、色彩混合与主题空间 (Color System & Surface Mix)

所有页面的色彩严禁使用非语义灰阶（如 text-gray-500），必须利用以下策略：

### 1. 基准主题色与派生色 (Semantic Bases)

- **Primary (粉红色系)** / **Secondary (浅蓝色系)** / **Tertiary (亮黄色系)**。
- 广泛使用 CSS Color-mix 打造的衍生背景色：例如 `--primary-muted` 和 `--surface-container`，为非交互状态提供极其柔和的底部色垫。
- 严禁用外显的大梯度渐变色 (Gradients)，以及擅自新增未包含在体系内的色值。

### 2. 半透明背景色块 (Translucent Block Stacking)

普遍是透明的，但是需要交互时，将引入色块

- 内容包裹容器：`bg-surface/50` (如 `about/index.vue`)，`bg-surface-variant/30`
- 媒体底边骨架：`bg-surface-variant/50` (如 `favorites.vue` 统一打底背景)。
- 活动色块：基于组件容器使用 `hover:bg-primary/5` 或 `hover:bg-on-background/5` 过渡填充。
- 文本辅助色：普遍使用 `text-on-background/60`（如小标题、说明文字） 或 `text-on-background/70`，最高到 `text-on-background/50` 约束低权级内容。

## 三、排版与字体基建 (Typography)

1. **全局字体**：全站通用字体的视觉基调依靠自定义的 **`荆南波波黑-Bold`** 支撑其可爱的形象。
2. **动态阅读降级**：
   在真正的主文字阅读区域（如 Markdown 正文、长数据列表），必须降级或回退为 **`ChillRoundF`** 配合系统无衬线体解决辨识度问题。
3. **文本截断机制**：列表或卡片如 `favorites.vue` 中的标题行均广泛使用流体约束 `line-clamp-1` 和 `line-clamp-2` 限制长度越界。
4. **字号层级策略**：通过 `clamp` 限定响应式缩放，标题不写死固定像素宽高。

## 四、全局动效与普遍交互 (Universal Motion & Interactions)

全站动画系统推崇 **短小、轻微、克制的色块反馈**，彻底摒弃以下行为：

- 严禁给卡片和模块增加**生硬跳动、过强的甩动弹跳感 (Bounce)**。
- 严禁悬浮时元素**上浮位移 (Translate-Y)**
- 严禁悬浮时元素**大幅度放大突出 (Scale)**，尽可能不使用。

### 1. 标准化响应式色彩切换 (Standard Hover & Active Colors)

本站的标准级底层悬浮交互模型非常统一，具备“微弱垫底 + 文字提亮”的原则：

- **一致性动画与过渡**：所有颜色切换（包括 `hover` 和 `focus`）强制通过 `transition-colors (或 transition-all) duration-200 ease-out` 进行控制。
- **业务主卡片悬浮**（如 `favorites.vue` 卡片容器与 `TimelineCard.vue` 容器）：底层悬停浸入极低频色块 `hover:bg-on-background/5` 或者 `hover:bg-primary/5`。
- **群体文案点亮**：容器悬挂 `group`，子级文本运用 `group-hover:text-primary` 控制点亮。
- **子级标签交互**（如 `TagList.vue`）：默认态以浅色填充 `bg-secondary/10 text-secondary` 作为衬托，悬浮后提档至 `hover:bg-secondary/20`。
- 只有极其简单的“底色溢淡 + 字体点亮”机制，**绝无特例的阴影放大、元素漂移和盒子突变**。这也是整个站点最核心且通用的 UX 心智。

### 2. 分页器控件降级与精简 (AnriPagination)

分页设计因地制宜，拥有自己独有的小巧思机制解决空间紧凑问题，没有沉重的全尺寸块：

- **`default` (常规平铺)**：常规的数字阵列加省略号扩展，选中项通过 `primary` 变现，其余为幽灵键 `ghost`。
- **`compact` (极简直达模式)**：这是专门为画廊（如 `favorites.vue` 角色区）准备的纵向截断模式。在布局内部利用原生的 `<input type="number">` 实现，结合屏蔽系统轮播箭头（`-webkit-appearance: none`）。直接让使用者通过 `@keyup.enter`、`@blur` 完成页码敲击。
- 其自身同样挂载绝对统一的颜色过渡策略：`focus:outline-none focus:ring-2 focus:ring-primary/20 transition-all duration-200`。

### 3. 媒体与面板空间过渡 (Transform & Pop)

- **图片的深度聚焦**：在 `AnriCharCard` 这类大视图模型中，长效聚焦感使用 `transition-transform duration-500 group-hover:scale-105` （时长高达半秒的镜头缓推）。
- **组件入退场 (Pop-ins)**：
  下拉框、图片面板必须使用非对称的时间落差：
    - 入场：`duration-200 ease-out` 配合 `scale-95 opacity-0 => scale-100 opacity-100`。
    - 退场：压缩至 `duration-150 ease-in` 快速清场。

### 3. 全局动画的错跑设计 (Out-of-Sync Animation)

樱花动效（`app.vue` / `.sakura`）并非无脑旋转循环，而是通过长达 `~30s` 的缓速、多对象时差组合和高达 `-21s` 的错落级延迟搭建，破除了常规的僵硬同频感知。

## 五、页面控件阵列与结构组装

### 1. 组件与图标 (Components & Icons)

- **公共组件层**：前缀 `Anri*` 是纯粹无业务语义的 UI 组件。严禁在其内部塞入业务判断、预置文案。严禁用它的时候在外面再套一层 `style` 进行二次覆盖或破坏其一致性。
- **图标**：全站必须统一使用 `Heroicons` 库，禁止随意引入或重复造属于其他风格的图标。业务相同逻辑已经在公共组件里写了，业务层禁止再起一套方案自己拼装。

### 2. 横纵架构与间距 (Layout)

- `layouts/default.vue` 的主容器统一通过 `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8` 来对版心建立网格收口。
- 分层列表：如 `favorites.vue` 则通常是 `grid grid-cols-2 md:grid-cols-3 lg:grid-cols-4 gap-4 md:gap-6`。

### 2. 标签与筛选分流 (Selectors / Nav Tabs)

如 `AnriSelector.vue` 与 `TagList` 所示的选项卡片理念：

- 激活态为高亮饱满色块 `bg-primary/10 text-primary`。
- 闲置与 Hover 态是 `hover:bg-on-background/10 hover:text-on-background` 加低权级原色 `text-on-background/60`，不越级宣兵夺主。

### 3. 缺省提示与加载形态 (Empty & Load states)

绝无庞大的占位骨架屏体系。

- 加载态直接由中心定调展示：`<AnriSpinner />`。
- 数据为空：`<div class="text-center py-12 text-on-background/50">无数据文案</div>`。
- 明确报错：`<ErrorDisplay />` 承接所有系统级别错误阻断。

### 4. 异步加载与动态分页交互心智 (Async Pagination & State Sync)

全站对于核心数据翻页和状态重载（以 `favorites.vue` 结合 `AnriPagination` 为标准）具备极其严格的**“原地等待后平滑过场”**规范，以杜绝传统 SPA 翻页时的生硬视野跳闪：

- **原地阻断挂载**：用户点击页码时，**页面原有内容绝对不做任何销毁、不出现白屏**。仅分页器组件通过自身属性传入 `:loading="loading"` 进入防护的加载状态。
- **时序拦截**：在数据接口未完全响应前（`await fetchData(page)`），界面必须保持绝对稳定。
- **渲染后丝滑归位**：数据就绪后，必须等到 DOM 真正渲染完毕（`await nextTick()`），再利用 `requestAnimationFrame` 执行视口跳转（如 `window.scrollTo({ top: 0, behavior: "smooth" })`），保障滚动平滑不突兀。
- **极简数字直达**：针对角色卡片等窄空间场景开启 `layout="compact"` 布局时，分页器会缩减为一个去除了上下滚动箭头的 `<input type="number">`，用户可以直接输入页码并通过 `@keyup.enter` 甚至 `@blur` 原地触发这套相同的重载跳转机制，精凑且高效。

## 六、严禁事项 (Absolute Prohibitions)

这是开发期的绝对铁律：

1. **绝对禁止“防炸”防御性编程**：不要为了所谓“稳一点”而编写兜底分支、默认吞错、静默失败！本站处在爆发期，必须进行破坏性重构；不要在组件里堆叠 `v-if="a?.b"` 的处理，绝不去兼容旧字段或旧参数，报错就暴露给 `ErrorDisplay` 解决。有旧代码干扰直接重构调用处，必须让代码呈现唯一真实逻辑。
2. **禁止自以为是与过度设计**：“可能以后用到”的抽象字段必须砍掉；未被验证的“也许以后会支持”严禁写进接口和类型中。
3. **禁止随意套用特例 UX**：不要把如 `ArticleBlock.vue` 的底层位移光晕套用到所有业务卡片。核心业务卡片的交互就是朴素的 `hover:bg-on-background/5` 过渡！
4. **禁止擅加虚拟业务文案**：禁止通过 AI 自动推测填加任何介绍、数据资源或未经确认的长文案占位。
5. **彻底禁用设计雷区**：“包含大投影、重度黑影”、“描边”、“渐变色”、“悬浮放大上移特效”、“荧光发光效果”、“不属于既定色系内的任何颜色值”、“用组件却再次覆盖样式造轮子”。
6. 禁止使用emoji
